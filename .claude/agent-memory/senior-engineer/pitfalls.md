## 2026-06-09 — ADR 0001 check-2 diff-shape proof not mechanizable from raw diff lines
Symptom: post-compaction verifier comparing `git diff HEAD` -/+ lines against the deleted entry band reported shape FAIL on all files despite a correct compaction. Root cause: git's diff algorithm matches blank lines and same-date `## 20..` headings between deleted entries and added ledger/compaction content as unchanged context, splitting hunks so extracted deletions never equal the band verbatim. Resolution: prove check 2 by byte-exact reconstruction instead — assert current file == `git show HEAD:<file>` with the selected band deleted, one compaction entry prepended below H1, and the terminal ledger section appended; this also proves surviving entries are byte-untouched. Also: `docket issue show <id> --json` nests fields under `.data` — read `.data.status`, not top-level `.status`, when verifying a close.

## 2026-06-10 — transcript-mining audit: multiline Bash `for s in $SKILLS` loops silently yield empty results
Symptom: per-skill audit loops (`for s in $SKILLS; do grep ... "$s" ...; done`) typed directly into the Bash tool printed the literal skill list on one line and 0/empty per-skill counts — three consecutive attempts produced no usable data with no error. Root cause: the harness/zsh handling of multiline compound commands with `$variable` interpolation inside nested `grep`/`python3` calls did not expand `$s` per-iteration as intended (the loop body collapsed). Resolution: for any per-item audit loop over a transcript/history corpus, Write the loop to a `.sh` file (`/tmp/claude/audit.sh`) and run `bash /tmp/claude/audit.sh` — single-line ad-hoc loops in the Bash tool are unreliable for this. Bonus: skill invocations in `*.jsonl` are recorded as `"name":"Skill","input":{"skill":"<name>",...}` — grep `"skill":"<name>"`; de-dup by distinct `sessionId` (subagent transcripts replicate ~10×); `is_error` counts are file-wide (whole-agent-run), NOT skill-tied — sample the error text before attributing any error to a skill.

## 2026-06-10 — ADR 0001 pitfalls-arm parity counter must skip surviving bullets between the prepended ledger and the first `## ` heading
Symptom: pitfalls compaction parity check (after-entries = before-entries − N) false-failed (reported 9 instead of 14) on staff-engineer's file even though the written file was correct. Root cause: the entry-counter detected the `## Harvested ledger (compacted)` section at the top and stayed in "skip-ledger" mode until the NEXT `## ` heading — but surviving `- ` bullet entries sit BETWEEN the prepended ledger section and the first `## ` heading, so they were wrongly skipped from the after-count. Resolution: count real entries by line shape directly (a real entry = `^## ` not `Harvested` OR `^- ` NOT starting `- [YYYY` ledger-bullet form); do NOT use a stateful "in-ledger until next ## heading" scan when the ledger is prepended above other bullet entries. Distinguish ledger bullets from real bullets by the `- [DATE]` prefix, not by position.

## 2026-06-11 — headless team-spawning `claude -p` never self-terminates; `git mv` rename invisible to plain `git diff`; BSD `head -n -1` illegal
Symptom (A): a `claude -p "/skill"` headless run whose skill spawns an in-process team (TeamCreate + Agent) emits a `"type":"result","subtype":"success"` event but the OS process NEVER exits — it loops re-emitting `result`/`init`, with the reconciler "Standing by for the shutdown confirmation event" and `TeamDelete` failing `Cannot cleanup team with N active member(s)`. Root cause: deadlock — the parent `-p` session blocks awaiting a shutdown-confirmation that only an interactive operator / external supervisor delivers, while the in-process teammate can't take a turn to approve its own shutdown. Resolution: for any orchestrator dispatching team-spawning headless sub-runs, an external wall-clock-cap-and-kill (Monitor timeout / pkill) is MANDATORY; exit-0 is NOT a usable pass signal — key off log-tail evidence (TeamCreate + final report) + worktree `status --porcelain` instead, and treat the kill signal as the expected terminal state. Note also: `--output-format stream-json` under `-p` REQUIRES `--verbose` (else exits 1 pre-execution: "stream-json requires --verbose"). Symptom (B): a reconcile sequence `git -C "$WT" diff -- <paths> | git apply` silently DROPPED a `git mv` rename. Root cause: `git mv` STAGES the rename, and plain `git diff` is unstaged-only, so the rename hunk never appears in the piped diff. Resolution: use `git -C "$WT" diff HEAD -- <paths>` (staged+unstaged vs base commit) so renames and any `git add`-ed changes are captured; plain `git diff` is unsafe for any reconcile that must survive staged edits. Symptom (C): `head -n -1 <file>` failed `head: illegal line count -- -1` on macOS. Root cause: GNU-only syntax; BSD `head` rejects negative counts. Resolution: reconstruct "all-but-last-line" via `git show HEAD:<file>` (the base version) or `sed '$d' <file>`.
